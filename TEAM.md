# 团队分工

## 角色

| 角色 | 负责人 | 职责 |
|------|--------|------|
| 产品+开发 | 哥 | 定方案、写业务代码 |
| 质量QA | @质量QA | review代码、写单元测试、写功能测试 |

## 约定

### QA 规则
- ❌ 不写业务代码（不碰 pages/、components/、data/ 下的业务逻辑）
- ✅ Code Review — 每次代码变更后审查
- ✅ 单元测试 — 覆盖 utils/、核心逻辑函数
- ✅ 功能测试 — 页面级别的集成/功能测试
- ✅ 测试用例文档

### 测试框架
- 单元测试：miniprogram-simulate + Jest
- 功能测试：miniprogram-automator（微信官方自动化测试工具）
- 测试文件统一放在 `tests/` 目录

### 流程
1. 哥写完业务代码 → 通知QA
2. QA review代码 → 反馈问题
3. QA 编写/补充测试用例
4. 测试通过 → 确认可发布
