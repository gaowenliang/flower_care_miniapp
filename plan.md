## 二、产品设计（2-3天）

### 2.1 核心功能定义
| 功能 | 优先级 | 说明 |
|------|--------|------|
| 我的花园 | P0 | 管理已添加的植物，查看状态 |
| 添加植物 | P0 | 拍照识别 / 搜索添加 |
| 浇水提醒 | P0 | 根据植物类型自动推算浇水周期 |
| 养护指南 | P0 | 每种植物的详细养护方法 |
| 养护日历 | P1 | 日历视图，查看养护计划 |
| 拍照识别 | P1 | 拍照识别植物种类 |
| 社区分享 | P2 | 用户分享养花经验 |
| 病害诊断 | P2 | 拍照识别植物病虫害 |

### 2.2 页面规划
1. **首页** — 我的花园概览，今日待办提醒
2. **添加页** — 拍照识别 / 搜索 / 分类浏览
3. **植物详情页** — 养护信息、养护记录、提醒设置
4. **日历页** — 养护日历，月视图
5. **我的页** — 个人设置、提醒管理

### 2.3 数据模型
```
Plant（植物模板）
├── id, name, latin_name
├── category（多肉/绿植/花卉/蔬果）
├── image_url
├── care_info（养护指南）
│   ├── light（光照需求）
│   ├── water（浇水频率）
│   ├── temperature（适宜温度）
│   ├── humidity（湿度）
│   ├── soil（土壤）
│   └── fertilizer（施肥）
└── tips（养护小贴士）

UserPlant（用户植物）
├── id, user_id, plant_id
├── nickname（自定义名称）
├── added_at, location（阳台/客厅/卧室）
├── last_watered, last_fertilized
└── photos[]（成长记录）

CareTask（养护任务）
├── id, user_plant_id
├── type（浇水/施肥/修剪/换盆）
├── frequency_days
├── next_date, last_done_date
└── enabled
```

## 四、技术要点

### 4.1 微信小程序基础
- 开发语言：WXML + WXSS + JavaScript（类似HTML+CSS+JS）
- 双线程模型：逻辑层（JSCore/V8）+ 渲染层（WebView）
- 不能直接操作DOM，用数据绑定驱动视图
- 包大小限制：主包 2MB，分包最大 20MB

### 4.2 关键API
- `wx.chooseMedia()` — 拍照/选图
- `wx.request()` — 网络请求
- `wx.requestSubscribeMessage()` — 订阅消息
- `wx.cloud.callFunction()` — 云函数调用
- `wx.setStorageSync()` — 本地存储

### 4.3 推荐后端方案
**方案A：微信云开发（推荐新手）**
- 免运维，自带数据库+存储+云函数
- 免费额度够小项目用
- 开发效率高

**方案B：独立后端**
- 更灵活，适合复杂业务
- Node.js (Express/Koa) 或 Python (FastAPI)
- 需要自己部署服务器

### 4.4 植物识别方案
| 方案 | 费用 | 精度 | 接入难度 |
|------|------|------|----------|
| 微信同声传译插件 | 免费 | 一般 | 简单 |
| 百度AI植物识别 | 500次/天免费 | 较好 | 简单 |
| 阿里云植物识别 | 按量付费 | 较好 | 简单 |
| Plant.id API | $0.002/次 | 很好 | 简单 |

### 4.5 养护数据来源
- 自建数据库（初期手工整理常见100种植物）
- 对接植物百科API
- 爬取公开养护指南（注意版权）
